# 1. Process Summary

## Purpose of the Process

The purpose of this process is to manage the complete lifecycle of a consulting engagement, beginning with a client inquiry and ending with successful project delivery and billing. The objective is to ensure consulting projects are delivered consistently while maintaining effective communication, standardized documentation, project visibility, and timely invoicing.

---

## Key Stakeholders Involved

| Stakeholder | Role / Responsibility |
|-------------|-----------------------|
| Managing Partner (Michael Reynolds) | Conducts discovery calls, qualifies opportunities, prepares proposals, assigns high-priority projects, and oversees client engagements. |
| Consultants | Conduct discovery calls, execute project work, communicate with clients, provide project updates, and submit time entries for hourly projects. |
| Senior Consultants | Prepare proposals independently and manage consulting engagements. |
| Project Manager | Creates projects in Asana, sets up shared folders, assigns tasks, manages project execution, and tracks consultant workload. |
| Accounting Team | Generates invoices based on project type and consultant time entries. |
| Clients | Share business requirements, review proposals, approve agreements, participate in project meetings, and receive project updates. |
